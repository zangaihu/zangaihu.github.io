java基础

面向对象与面向过程

面向过程

概述：自顶向下的编程模式

利用一个个函数编程，依次调用，各种if-else和for.

面向对象

概述：过程高度抽象化

将具有共同特性的事物抽象成一个个类

三大特性: 封装、继承、多态

  • ==封装==：将具有共同特征的事物进行高度抽象
  • ==继承==：继承父类，可以保留父类的特征，并扩展自己的特征。分为实现继承和接口继承。
  • ==多态==：向上转型、向下转型。传入参数为子类实例，会执行子类同名方法。

六大原则

  1. 单一职责原则：每个类只负责一个单一的职责，有且只会有一个原因引起类的变化。专注，也是一种品质。
  2. 开放封闭原则：对扩展开放，对修改关闭。
  3. 里氏替换原则：任何基类出现的地方，子类一定可以出现。
  4. 依赖倒置原则: 针对接口编程，依赖与抽象而不依赖于具体。
  5. 迪米特原则：一个类应尽量少的与其他类发生相互关系，使模块功能相对独立。
  6. 接口隔离原则：不要使用一个总接口，使用多个隔离的小接口，降低耦合。
